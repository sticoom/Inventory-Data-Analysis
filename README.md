# 📚 业务规则字典 (数据清洗与运算核心)

### 一、 底层映射与清洗逻辑 (阶段 2 严格执行)

#### 1. FNSKU 唯一映射法则 (防报错处理)
- 《销量统计》和《产品运营信息表》中可能存在重复的 `FNSKU`。在建立映射字典时，**必须使用 `.groupby('FNSKU').first()`** 进行去重提取，确保字典的键绝对唯一，防止后续映射报错。
- **信息补全**：利用上述构建的唯一 FNSKU 字典，对《销量预测表》中缺失的 `类目`、`国家`、`SKU`、`产品定位(标签)` 进行全量映射回填。

#### 2. 预测表日期解析与“PD”列强归集
预测表的动态列名可能是“3月16日”、“3月23日”，或者包含特定大促标识如“6月PD”。
- **正则提取与映射**：遍历预测列，利用正则提取月份数字。如“3月16日” -> 对应 `2026-03`。
- **大促(PD)归集**：若列名包含“PD”（Prime Day），例如“6月PD”，提取其前缀数字“6”，**强制归入** `2026-06`。
- **行级横向聚合**：同属一个 `YYYY-MM` 的所有列数值，在单行（单 SKU）维度内使用 `.sum(axis=1)` 进行横向加总，生成该 SKU 唯一的月度预测值。

#### 3. 大盘走势表的“类目”模糊匹配 (降噪清洗)
《大盘趋势表》的类目名称与本店不完全一致，需进行正则截断后再匹配。
- **清洗逻辑**：去除大盘类目字符串末尾的 `收纳`、`品线`、`系列` 等无意义后缀。
  *(例如：“下水槽收纳品线” -> 清洗为“下水槽”)*。清洗后方可与本店销售数据的主表进行 Merge。

#### 4. 国家编码标准化与兜底
通过 Pandas 的 `.map()` 或自定义函数对国家列进行映射。
- **转换规则**（忽略大小写）：日本->`JP` | 加拿大->`CA` | 美国->`US` | 墨西哥->`MX` | 澳洲->`AU` | 德国->`DE` | 英国->`UK`
- **兜底规则**：若值不在上述列表中（如法国、西班牙、空值），一律默认返回 `DE`。

### 二、 AI 智能排雷算法 (阶段 4 严格执行)

对于页面上渲染的每一个聚合图表（如某个选定品类的汇总数据），必须在其后执行以下三大算法：

#### 算法 1：历史月均及格线与极值提取
- 提取该维度下 2024 年(1-12月)和 2025 年(1-12月)所有大于 0 的实际销量数据。
- 计算平均值，得出**“历史月均线 (Baseline)”**。
- 提取这 24 个月中的绝对最大值和最小值所在的年月，用于前端绘制红绿极值点。

#### 算法 2：3个月滑动平均 (找综合淡旺季)
- **预处理**：`月度绝对平均年[m] = (2024年[m]销量 + 2025年[m]销量) / 2`，生成 1-12 月的基准数组。
- **滑动窗口**：构建循环数组，计算 `(上月 + 本月 + 下月) / 3`。（注：1月的前一月是12月，需处理跨年索引）。
- **极值捕获**：找出滑动平均值最大的连续三个月定义为**“综合连续旺季”**；最小的连续三个月定义为**“综合连续淡季”**。

#### 算法 3：3:7 加权基准线排雷 (断货/滞销预警)
针对选定的预测期（如 2026年3月-7月）：
- **加权基准值 (Weighted Baseline)** = `Σ (2025年同期各月销量 × 0.7) + Σ (2024年同期各月销量 × 0.3)`
- **偏差率 (Deviation)** = `(未来对应月份的预测总销量 - 加权基准值) / 加权基准值 * 100%`
- **诊断触发机制 (卡片输出)**：
  - `偏差率 > +20%` → **⚠️ 滞销/超备风险**（附文案：“当前预测偏离加权基准 +XX%。请结合产品定位确认，若非爆发期新品，请防范库存积压。”）
  - `偏差率 < -20%` → **🚨 断货/保守风险**（附文案：“当前预测偏离加权基准 -XX%。相较往年过于保守，请防范断货流失坑位。”）
  - `-20% <= 偏差率 <= +20%` → **✅ 备货健康**（附文案：“预测贴合历史加权大盘走势。”）
